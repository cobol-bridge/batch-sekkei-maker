---

## スライド2：課題

### 多くのCOBOL現場が抱える問題

| 課題 | 現状 |
|------|------|
| 設計書がない | 過去の改修でドキュメント作成が省略された |
| 設計書が古い | 現物コードと乖離していて信用できない |
| 作成工数が大きい | 読み解き＋転記で1本あたり半日〜1日 |
| 移行・保守に支障 | 担当者の属人的な知識に依存している |

> 金融・保険・公共系では、COBOLバッチが数十年にわたり稼働しているケースも多く、ドキュメント整備は慢性的な課題となっています。

---

## スライド3：解決策

### バッチ設計書メーカー

```
COBOLソースファイルを選択
        ↓
「設計書を生成する」をクリック
        ↓
     数秒で完成
```

**自動解析する項目：**
- I/O定義（ファイル名・編成・レコード長・物理ファイル名）
- 処理フロー（PERFORM呼び出し階層）
- 例外処理（INVALID KEY / AT END / ON SIZE ERROR）

---

## スライド4：最大の特徴

### 完全オフライン動作

```
❌ クラウドAIサービス
　→ ソースコードを外部サーバーに送信
　→ 情報セキュリティポリシーに抵触するリスク

✅ バッチ設計書メーカー
　→ すべてローカルPC内で処理
　→ インターネット接続不要
　→ 社外秘コードを一切外部送信しない
```

**金融・公共系の情報セキュリティ要件をクリア**

## スライド5：導入効果（試算）

### 設計書作成工数の削減

| 作業 | 従来 | 本ツール |
|------|------|---------|
| 1本あたりの作成時間 | 4〜8時間 | 数秒（たたき台生成） + 1〜2時間（確認・加筆） |
| 50本の場合 | 200〜400時間 | 50〜100時間 |
| **削減効果** | | **約75%削減** |

> ※上記は試算値です。ソースの複雑度・担当者スキルにより変動します。
